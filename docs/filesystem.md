# 文件系统介绍

## 概述

文件系统是操作系统用于管理存储设备上数据的机制，它定义了数据如何存储、组织和访问。

## Windows 主要文件系统

### FAT32 (File Allocation Table 32)

**特点**
- 最大卷大小: 2TB (理论)，实际建议 32GB
- 最大文件大小: 4GB - 1 字节
- 兼容性最好，支持所有操作系统

**结构**
```
┌─────────────┬─────────────┬─────────────┬──────────────┐
│ 引导扇区    │ FAT 表 1    │ FAT 表 2    │ 数据区       │
│ (Boot)      │ (Primary)   │ (Backup)    │ (Data)       │
└─────────────┴─────────────┴─────────────┴──────────────┘
```

**FAT 表项含义**
| 值 | 含义 |
|----|------|
| 0x00000000 | 空闲簇 |
| 0x0FFFFFF7 | 坏簇 |
| 0x0FFFFFF8-0x0FFFFFFF | 文件结束标记 |
| 其他值 | 下一个簇号 |

**数据恢复要点**
- 删除文件时仅将 FAT 表项置零
- 目录项中文件名首字节改为 0xE5
- 簇链断裂导致数据丢失

---

### NTFS (New Technology File System)

**特点**
- 最大卷大小: 256TB
- 最大文件大小: 16EB
- 支持文件权限、加密、压缩、日志

**核心结构**
```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│ 引导扇区     │ MFT 区域     │ 系统文件     │ 数据区       │
│ (Boot)       │ (MFT)        │ (System)     │ (Data)       │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

**MFT (Master File Table) 记录结构**
```
MFT 记录 (通常 1024 字节)
┌─────────────────────────────────────────────────────────┐
│ MFT 头部 (48 字节)                                      │
│  - 签名: FILE                                           │
│  - 更新序列号                                            │
│  - 记录使用状态                                          │
├─────────────────────────────────────────────────────────┤
│ 属性 1: $STANDARD_INFORMATION                           │
│  - 创建时间、修改时间、访问时间                          │
│  - 文件属性 (只读、隐藏、系统等)                        │
├─────────────────────────────────────────────────────────┤
│ 属性 2: $FILE_NAME                                      │
│  - 文件名 (Unicode)                                     │
│  - 父目录引用                                           │
├─────────────────────────────────────────────────────────┤
│ 属性 3: $DATA (文件数据)                                │
│  - 小文件: 数据直接存储在 MFT 中 (Resident)             │
│  - 大文件: 数据运行指向外部簇 (Non-Resident)            │
├─────────────────────────────────────────────────────────┤
│ 属性 N: ...                                             │
└─────────────────────────────────────────────────────────┘
```

**数据运行 (Data Runs) 格式**
```
[大小字节数][偏移字节数][大小值][偏移值]

示例: 21 03 45 01
  - 2: 大小字段占 2 字节
  - 1: 偏移字段占 1 字节
  - 03 45: 大小 = 0x0345 簇
  - 01: 偏移 = 0x01 簇
```

**关键系统文件**
| 文件名 | 作用 |
|--------|------|
| $MFT | MFT 本身 |
| $MFTMirr | MFT 前几条记录的备份 |
| $LogFile | 事务日志 |
| $Volume | 卷信息 |
| $AttrDef | 属性定义 |
| $Bitmap | 簇使用位图 |
| $Boot | 引导扇区 |
| $BadClus | 坏簇列表 |

**数据恢复要点**
- 删除文件时 MFT 记录标记为"未使用"
- 文件名和时间戳信息通常保留
- 小文件可直接从 MFT 恢复
- 大文件需要解析数据运行定位簇

---

### exFAT (Extended File Allocation Table)

**特点**
- 最大卷大小: 128PB
- 最大文件大小: 16EB
- 专为闪存设计 (U盘、SD卡)

**结构**
```
┌─────────────┬─────────────┬─────────────┬──────────────┬──────────────┐
│ 引导扇区    │ 备份引导    │ FAT 表      │ 数据位图     │ 数据区       │
│ (Boot)      │ (Backup)    │ (FAT)       │ (Bitmap)     │ (Data)       │
└─────────────┴─────────────┴─────────────┴──────────────┴──────────────┘
```

**与 FAT32 的区别**
- 支持大于 4GB 的文件
- 使用位图而非 FAT 表管理空闲空间
- 目录项支持更多元数据

---

## 文件系统对比

| 特性 | FAT32 | NTFS | exFAT |
|------|-------|------|-------|
| 最大文件 | 4GB | 16EB | 16EB |
| 最大卷 | 2TB | 256TB | 128PB |
| 日志 | 否 | 是 | 否 |
| 权限 | 否 | 是 | 否 |
| 压缩 | 否 | 是 | 否 |
| Windows | 全版本 | NT+ | Vista+ |
| macOS | 读写 | 只读 | 读写 |
| Linux | 读写 | 读写 | 读写 |
| 适用场景 | U盘/小设备 | 系统盘 | 大容量闪存 |

---

## 数据恢复优先级

1. **NTFS** - 最常见，MFT 结构利于恢复
2. **exFAT** - 闪存设备常见
3. **FAT32** - 旧设备兼容
